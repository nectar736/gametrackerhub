# AGENTS.md - Board Game Tracker

## Project Overview

A static board game tracking application using 11ty with localStorage persistence. The source files are in `src/` and build to `_site/`.

## Commands

```bash
npm run build     # Build the site with Eleventy
npm run dev       # Build and serve with live reload on port 3000
npm run start     # Alias for dev
```

There are no automated tests or linting configured. Run `npm run build` after changes and verify the site works in a browser.

## Code Style Guidelines

### General

- Use IIFE with `'use strict'` for main app.js to avoid global scope pollution
- Modules (analytics.js, export.js, storage.js) use ES6 classes
- Target ES6+ browsers - no transpilation required

### File Structure

```
src/
├── index.njk          # Main page (Eleventy template)
├── _layouts/base.njk # Base HTML layout
├── css/style.css     # Styles
└── js/
    ├── app.js        # Main application (IIFE pattern)
    ├── storage.js   # Storage module (ES6 class)
    ├── analytics.js # Analytics module (ES6 class)
    └── export.js    # Export module (ES6 class)
```

### Naming Conventions

- **Files**: kebab-case (e.g., `storage.js`, `analytics.js`)
- **Classes**: PascalCase (e.g., `Storage`, `Analytics`)
- **Methods/Variables**: camelCase (e.g., `getAllUsers()`, `currentUser`)
- **Constants**: UPPER_SNAKE_CASE (e.g., `STORAGE_KEYS.USERS`)
- **Storage Keys**: Prefix with `bgt_` (e.g., `bgt_users`, `bgt_games`)

### Code Formatting

- Indentation: 2 spaces
- Object braces on same line: `if (condition) {`
- Use semicolons after statements
- Multi-line conditionals for complex if statements
- Group related constants at top of file

### Imports/Exports

- Use `window.ClassName` pattern for global exports
- Modules export to window: `window.Storage = new Storage();`
- Classes accept optional storage dependency: `constructor(storage = window.Storage)`

### Error Handling

- Wrap localStorage operations in try/catch
- Log errors with `console.error()`
- Return safe defaults on error (empty array `[]` for reads, `false` for writes)
- Use `confirm()` for destructive actions, `alert()` for notifications

### JSDoc

- Add file-level JSDoc comment at top of each module
- Describe purpose and main functionality

Example:
```javascript
/**
 * Storage Module - Handles localStorage operations for the board game tracker
 * Provides a unified interface for CRUD operations on all data types
 */
```

### Patterns

**Storage Keys** - Define as constant object at module top:
```javascript
const STORAGE_KEYS = {
  USERS: 'bgt_users',
  GAMES: 'bgt_games',
  // ...
};
```

**Class Methods** - Use underscore prefix for private methods:
```javascript
_getData(key) { ... }
_setData(key, data) { ... }
```

**ID Generation** - Use timestamp + random:
```javascript
generateId() {
  return Date.now().toString(36) + Math.random().toString(36).substr(2, 9);
}
```

### Build Process

- Edit source files in `src/` directory
- `_site/` is generated by Eleventy - do not edit directly
- After modifying source files, run `npm run build` to regenerate `_site/`
- Both `src/js/app.js` and `_site/js/app.js` may need updates (some users run from _site)

### Testing

Manual testing required:
1. Run `npm run dev`
2. Open http://localhost:3000
3. Test core functionality: add user, add game, record match, view analytics
4. Verify Settings > Clear All Data works without re-seeding

### Common Issues and Solutions

**Clear All Data not working**: When clearing localStorage, ensure `bgt_seeded` flag is preserved or reset to prevent automatic re-seeding:

```javascript
clearAllData() {
  if (confirm('Are you sure you want to delete ALL data? This cannot be undone!')) {
    localStorage.clear();
    localStorage.setItem('bgt_seeded', 'true'); // Prevent re-seeding
    Storage.init();
    this.render();
  }
}
```

**Syntax errors in reduce/map**: Always close braces and parentheses properly:

```javascript
// Wrong:
const totalScore = matches.reduce((sum, m) => {
  return sum + (result?.score || 0);
0);  // Orphaned!

// Correct:
const totalScore = matches.reduce((sum, m) => {
  return sum + (result?.score || 0);
}, 0);
```
